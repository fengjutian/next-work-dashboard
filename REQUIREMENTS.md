# AI 提示词助手 — 需求文档

## 1. 项目概述

### 1.1 项目名称
**next-work-dashboard**（暂定）— 基于 Electron 的 AI 提示词注入桌面应用。

### 1.2 项目定位
一个 Electron 桌面应用，内置 WebView 浏览器，可打开 DeepSeek、ChatGPT、Kimi 等 AI 对话网站，并支持将预设提示词一键注入到网页的输入框中，提升 AI 交互效率。

### 1.3 目标用户
- 高频使用多个 AI 对话平台的用户
- 需要复用标准化提示词的场景（如客服话术、代码审查、文档生成等）
- 希望统一管理提示词库的个人或小团队

## 4. 浏览器 / 注入模块（核心）

### 4.1 WebView 浏览器
| 编号 | 需求 | 说明 |
|------|------|------|
| B01 | 多标签页 | 支持多个 WebView 标签页，每个标签打开一个 AI 站点 |
| B02 | 标签页管理 | 新建、关闭、切换、拖拽排序标签页 |
| B03 | 页面导航 | 支持前进、后退、刷新、回到首页 |
| B04 | 持久化 Session | 关闭标签后保留登录态（Cookie/LocalStorage 持久化） |

### 4.2 提示词注入引擎
| 编号 | 需求 | 说明 |
|------|------|------|
| B05 | 注入触发 | 通过侧边栏点击提示词、快捷键、右键菜单三种方式触发注入 |
| B06 | DOM 选择器注入 | 根据站点配置的 CSS 选择器定位输入框，通过 preload 脚本将提示词写入 |
| B07 | 注入状态反馈 | 注入成功/失败给出 Toast 提示；失败时提示可能原因（选择器不匹配等） |
| B08 | 模拟输入 | 注入时触发 Input/Change 事件，确保 AI 网站的框架（React/Vue）能感知输入变化 |
| B09 | 自动发送 | 当注入模式为「填充并发送」时，注入后自动查找发送按钮并触发点击 |

### 4.3 侧边栏 / 浮动面板
| 编号 | 需求 | 说明 |
|------|------|------|
| B10 | 侧边栏 | 应用左侧固定侧边栏，展示提示词列表、分类树、搜索框 |
| B11 | 侧边栏折叠 | 侧边栏可折叠，最大化 WebView 显示面积 |
| B12 | 浮动快捷面板 | 全局快捷键唤出浮动搜索面板（类似 Spotlight/Alfred），搜索提示词后回车注入 |

## 5. 技术架构（概要）

### 5.1 技术栈
| 层 | 技术选型 |
|----|----------|
| 框架 | Electron 28+ |
| 渲染进程 | React 18 + TypeScript |
| UI 组件库 | shadcn/ui + Tailwind CSS |
| 状态管理 | Zustand |
| 持久化 | sql.js + drizzle-orm |
| 构建工具 | electron-forge + Vite |
| 代码规范 | ESLint + Prettier |

### 5.2 进程架构
```
┌─────────────────────────────────────┐
│            主进程 (Main)              │
│  · 窗口管理   · 全局快捷键             │
│  · 数据持久化 · IPC 通信              │
│  · 菜单/托盘  · 自动更新              │
└──────────┬──────────────────────────┘
           │ IPC
┌──────────▼──────────────────────────┐
│         Preload 脚本                 │
│  · contextBridge 暴露安全 API        │
│  · DOM 操作注入能力                  │
└──────────┬──────────────────────────┘
           │
┌──────────▼──────────────────────────┐
│          渲染进程 (Renderer)          │
│  · React UI (设置/提示词管理)         │
│  · WebView 标签页容器                │
│  · 侧边栏/浮动面板                   │
└─────────────────────────────────────┘
```

### 5.3 安全模型
- WebView 中运行的 AI 网站与主应用隔离
- 通过 `contextBridge` 暴露受控 API，不直接暴露 Node.js 能力给 WebView
- 注入操作由 Renderer 发起 → Preload 脚本在 WebView 中执行 DOM 操作
- 提示词数据仅存储在本地，不上传任何服务器

## 9. 已决策项

| 事项 | 决策 | 理由 |
|------|------|------|
| UI 组件库 | **shadcn/ui + Tailwind CSS** | 紧凑桌面风格、暗色模式零成本、浮动面板定制灵活、按需复制体积小 |

| 构建工具 | **electron-forge + Vite** | 官方推荐，插件生态完善（打包/更新/签名），和 Vite 集成成熟 |
| 托盘 | **需要** | 关闭窗口最小化到托盘，后台运行，托盘菜单快速唤起/退出 |
| WebView | **原声 `<webview>` 标签** | 多标签管理更自然，partition 持久化登录态，相比 BrowserView 开发成本低 |

## 10. 竞品分析

### 10.1 市场格局

```
                  走API模式              走网页DOM注入
              ┌──────────────┐       ┌──────────────┐
  桌面应用     │ ChatBox       │       │ next-work-dashboard    │  ← 唯一占据
              │ Jan.ai        │       │ (我们)        │     此格
              │ LM Studio     │       │              │
              └──────────────┘       └──────────────┘
              ┌──────────────┐       ┌──────────────┐
  浏览器扩展   │    不存在      │       │ AIPRM        │
              │              │       │ Prompt Genius│
              └──────────────┘       └──────────────┘
```

### 10.2 逐类分析

#### 桌面应用

| 产品 | 交互方式 | 与我们的差异 |
|------|----------|-------------|
| **ChatBox** | API 直连，用户填 Key | 纯 API 客户端，无提示词管理，不保留官网体验 |
| **ChatALL** | 多站点 WebView，统一输入框 → 广播 | 一对多广播，非提示词库注入；站点列表固定 |
| **Jan.ai** | 本地推理 + 远程 API | 偏向 Ollama 类本地运行，无注入逻辑 |
| **LM Studio** | 本地模型推理引擎 | 纯本地推理，不涉及网页 |

> **结论**：现有桌面应用全是"AI 客户端"，没有一个是"在官网 WebView 中注入提示词"的模式。

#### 浏览器扩展

| 产品 | 注入原理 | 优势 | 局限 |
|------|----------|------|------|
| **AIPRM** | Content Script 操作 ChatGPT DOM，插入提示词 | 4000+ 社区提示词、团队共享 | 仅 ChatGPT/Claude；有付费墙 |
| **ChatGPT Prompt Genius** | Content Script + 侧边栏，模板填入 | 变量支持、历史导出 | 仅 ChatGPT |
| **Superpower ChatGPT** | DOM 注入增强 UI | 对话管理 + 快捷输入 | 已下架 |

> **技术相同，载体不同**：扩展用 Content Script 注入，我们用 Preload 注入——本质是同一类方案。但扩展只能绑一个浏览器，next-work-dashboard 是独立桌面应用。

#### Web 平台

| 产品 | 类型 | 说明 |
|------|------|------|
| **PromptBase** | 提示词交易市场 | 买卖提示词（310k+ 条），类 Etsy |
| **FlowGPT** | 提示词社区 | 分享 + 在线运行 |
| **TypingMind** | Web Chat UI | 自带 API Key 的前端，内置提示词库 |

> 这些不涉及注入，是独立的提示词社区或聊天前端。

### 10.3 next-work-dashboard 差异化定位

| 维度 | next-work-dashboard | ChatBox | AIPRM |
|------|-----------|---------|-------|
| 需要 API Key | ❌ 不需要 | ✅ 必须 | ❌ 不需要 |
| 保留官网体验 | ✅ 完整保留 | ❌ 无 | ✅ |
| 多AI站点 | ✅ 任意可配置 | 多模型API | ❌ 仅 ChatGPT |
| 提示词管理 | ✅ 本地 SQLite | ❌ | ✅ 社区+私有 |
| 变量模板 | ✅ | ❌ | ✅ |
| 数据隐私 | ✅ 纯本地 | ⚠️ API传输 | ✅ |
| 全局浮动面板 | ✅ Alfred 风格 | ❌ | ❌ |

### 10.4 风险与对策

| 风险 | 应对 |
|------|------|
| AI 网站更新 HTML → 选择器失效 | 用户可自行更新 CSS 选择器（S02/S03）；后期可建社区适配仓库 |
| 多 WebView 内存占用大 | 延迟加载未激活标签页；限制同时打开标签数 |
| 桌面应用安装门槛 > 扩展 | 提供免安装便携版；后期考虑 Chrome 扩展作为轻量入口 |

### 10.5 定位口号

> "不是又一个 ChatGPT 客户端——是你所有 AI 网站的提示词遥控器"
